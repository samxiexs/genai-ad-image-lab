# Product-oriented Function v2 Advertising Image Prompt 中文说明

## 共享实验指令
你正在为消费者广告实验生成受控刺激材料。

请把提供的白底商品图作为准确的源商品。保持商品身份不变，保留商品的核心形状、材质、颜色、包装、logo、可见设计和物理结构。

不要虚构商品功能、成分、认证、奖项、功效声明、口号、评分、价格、二维码、徽章、图表、箭头或品牌资产。

不要在图中生成新的可读文字。源商品图中原本已有的文字、logo 或包装信息可以保留，但不要新增广告文案，也不要重写包装文字。请保留干净留白，方便后续用受控模板叠加文字。

避免截图感、横幅模板感、低质量拼贴、表情包风格、杂乱电商 mockup、畸形手部、破损包装、乱码字符和不真实的商品几何结构。

## 生成 Product-oriented Function v2 广告图

主要说服路径：具体功能效用、产品如何运作、问题-解决价值，以及商品如何帮助用户的可见证据。图像必须传达商品“做什么”或“能实现什么”，而不只是传达它高级、好看、有情绪、象征性强或适合送礼。

### 广告机制
图像应该帮助观看者快速判断商品的实际 job-to-be-done、关键功能 affordance、可见操作部件、握持或使用逻辑、兼容性、适配、安装、收纳、保护、新鲜度、护理、性能、耐用或易用性。请使用商品标题、品牌、类目、价格和促销元数据推断合理功能线索，并把这些线索转译成非文字视觉证据。

### 商品元数据
- 标题：{ori_title}
- 品牌：{creative_id_brand}
- 类目：{level_one_category_name}
- 价格：{creative_id_price}
- 促销：{creative_id_promotion}

### 功能推断指令
- 先从标题和类目推断商品类型、实际用户问题、可能使用任务、材质或机制，以及消费者购买它的主要功能理由。
- 对电子、工具、家电、运动、家居、美容仪器、玩具、宠物、食品、药品、图书、配饰、服饰、珠宝和礼品等类目，推断适合该品类的功能证据，例如操作、界面、组装、适配、扣合、便携、收纳、清洁、保护、兼容、新鲜度、涂抹/使用、整理、舒适、耐用或护理。
- 对装饰性、穿戴性、奢侈品或礼赠商品，如果机械功能有限，请把功能理解为适配、扣合、尺度、穿戴支撑、安全收纳、材质处理、包装保护、搭配、护理或礼赠准备度。不要把这些商品变成地位、华丽、浪漫或抽象向往图。
- 不要虚构标题、类目或源图无法支持的功能。不确定时，使用通用品类 affordance，而不是创造假功能。

### 视觉方向
- 商品必须是主导主体，并且至少呈现两个具体功能证据线索。
- 优先使用受控的产品功能演示构图：中性桌面、干净棚拍工作台、有序部件关系、真实开合状态、可见机制、尺度参照、配件关系、安全收纳、材质对照，或只用于说明握持、适配、扣合、涂抹或尺度的手模。
- 只有当道具能够解释功能时才使用相关中性道具。道具必须像证据，而不是生活方式装饰或象征氛围。
- 允许出现一个整合在商业产品摄影中的局部细节区，但不能有标签、箭头、标注或文字。
- 光线、阴影、反射和背景应帮助说明形状、材质、操作部件、affordance 和可用性。美感只有在提升功能理解时才成立。
- 避免把纯白或近白 packshot 作为主导视觉方案。图像必须比更干净的商品图提供更多功能信息。
- 确保画面与商品标题、类目、价格水平和促销语境一致。
- 保留干净留白，方便后续统一叠加受控文字。

### 严格边界
- 不要创造生活方式故事、社交场景、浪漫场景、高级 mood 肖像或抽象象征构图。
- 不要让美感、高级感、礼赠情绪、氛围或品牌人格成为主要说服理由。
- 不要把视觉隐喻作为主导手段。功能必须通过具体产品证据可见。
- 不要新增文字、口号、假标签、假徽章、认证、价格、评分、图表、箭头或对比声明。
- 不要虚构商品功能、成分、奖项或品牌信息。
- 不要改变商品形状、颜色、logo、包装或物理结构。
- 如果生成图主要会被描述为高级静物、象征 mood 图、生活方式场景，或同一张 packshot 换了更好光线，则视为失败。

### 验证标准
观看者应能在“这张图帮助我理解产品做什么”和“这张图让产品功能价值很清楚”上给高分。图像应通过 3 秒功能测试：从未听说过该商品的人应能快速理解它支持的实际任务、affordance 或功能收益，同时不会主要把它理解为生活方式想象图或象征品牌 mood 图。

### 输出目标
生成一张广告图，首要目标是功能效用和可见产品功能。美感应该服务于功能理解，而不是替代功能理解。
