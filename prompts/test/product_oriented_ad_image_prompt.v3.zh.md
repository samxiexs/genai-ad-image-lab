# Product-oriented Advertising Image Prompt v3 中文说明

## 共享实验指令
你正在为消费者广告实验生成受控刺激材料。

请把提供的白底商品图作为准确的源商品。保持商品身份不变，保留商品的核心形状、材质、颜色、包装、logo、可见设计和物理结构。

不要虚构商品功能、成分、材质、认证、奖项、功效声明、口号、评分、价格、二维码、徽章、图表、箭头或品牌资产。

不要在图中生成新的可读文字。源商品图中原本已有的文字、logo 或包装信息可以保留，但不要新增广告文案，也不要重写包装文字。请保留干净留白，方便后续用受控模板叠加文字。

避免截图感、横幅模板感、低质量拼贴、表情包风格、杂乱电商 mockup、畸形手部、破损包装、乱码字符、不真实的商品几何结构和未经支持的人群假设。

## 生成 Product-oriented 广告图

### Brand concept 定义
依据 Park, Jaworski, and MacInnis (1986)，brand concept 是企业选择的、来自消费者基本需求的品牌意义。本 prompt 对应 functional brand concept：将品牌或商品与外部产生的消费需求相连，例如解决当前问题、预防潜在问题、降低使用阻力、支持某个任务或让消费过程更可靠。

### 主要说服路径
图像应该让商品的问题解决角色清楚可见。观看者应能推断商品帮助解决什么消费问题或实践任务。不要让象征身份、生活方式向往、感官愉悦、新奇感或抽象氛围成为主要说服理由。

### 商品元数据

- 品牌：{creative_id_brand}
- 类目：{level_one_category_name}
- 价格：{creative_id_price}
- 促销：{creative_id_promotion}

### Concept-image linkage
- 先只根据输入推断商品类型、可能任务、可见结构、相关 affordance 和合理消费问题。
- 将 functional concept 转译为一般性的视觉证据，不写未经支持的功能声明。
- 可以使用商品使用关系、任务语境、握持线索、操作线索、适配线索、保护线索、收纳线索、应用线索、兼容线索或其他品类合适的证据，只要与源商品和元数据一致。
- 商品必须是视觉锚点。任何道具、手部、表面或背景线索都必须解释实际问题或任务，不能变成生活方式装饰。
- 不要默认某个品类天然就是 functional；只有图像把商品和实践消费需求相连时，它才是 functional 呈现。

### 生成自由度
模型可以选择构图、角度、背景、光线和风格，但只能在它们支持 functional concept 且不违背源商品或元数据时使用。优先使用概括、可辩护的线索，不堆砌细碎虚构场景。

### 严格边界
- 不要把象征身份肖像、地位 mood、浪漫场景、社交归属场景或抽象向往图作为主导印象。
- 不要让感官愉悦、新奇、幻想或沉浸成为主视觉信息。
- 不要使用标签、箭头、字幕、图表、前后对比文字、比较声明、假徽章、假认证或新增可读文字。
- 不要虚构输入未支持的机制、成分、产品表现、奖项或品牌信息。
- 不要改变商品形状、颜色、logo、包装或物理结构。

### 验证标准
观看者应能在“这张图帮助我理解商品解决什么实际问题”和“这张图让商品的功能价值清楚可见”上给高分。图像应通过 3 秒 functional concept 测试：观看者应快速推断实践任务或消费问题，而不是主要把它理解为象征身份表达或体验刺激。

### 输出目标
生成一张广告图，首要目标是 functional brand concept 和 product-problem linkage。美感应服务于功能价值理解，而不是替代它。
